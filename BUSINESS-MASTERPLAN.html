The universal loop and the four gates

Every automation in all three engines is the same machine: signal → skill classifies & drafts → gate → action → audit → precision score → autonomy. Gate levels are declared per skill and graduate per client as measured precision earns it:

GateMeaningExamples
G0 silentActs alone, internal output onlyentity matching, flag ranking, lead scoring, competitor diffs, sentiment
G1 auto + visibleActs alone, logged, human can undotask creation ≥0.8 conf, chases, follow-up lists, provisioning, invoices
G2 approve firstHuman clicks before anything client-facing or spend-changingclient emails, Ads mutations, publishing, proposals, concepts, translations
G3 restrictedNamed senior human; AI may not even draftcomplaints/cancellations (Wally, no AI ack — ever), high-risk Ads structure, refunds, offboarding

G2 → G1 graduation: a category unlocks auto-send for a client after a measured streak (e.g. 50 completion emails sent unedited). A human flips the switch, informed by the precision-ledger number — never automatic. The ahpra-checker is outside this system entirely: a hard blocking gate on all client-facing content, agency- or client-initiated, forever.

06

How the skills get built — the anti-slop authoring system

Format & runtimes

Every skill is authored in the Agent Skills format — a folder with SKILL.md + resources at /skills/<name>/. The same folder runs in Claude Code during development, and in production via the API (the hosted Skills API provides create/version/list — exactly our versioning model). Pre-built Anthropic skills (xlsx · docx · pptx · pdf) handle document output inside report-writer, proposal-writer, and audit exports — buy the mechanical layer, build the judgement layer.

The raw material already exists

The intranet SOPs + ~40 YouTube training videos transcribe into skill drafts. The support@ sent folder is years of labelled triage decisions in the team's actual voice. Past reports, diagnoses, proposals, and live sites are input→output pairs for free. No skill is written from imagination — each distils work the team already does well.

The per-skill recipe (eval-first)

  • Collect 30–50 real historical examples
  • Split: golden set (held-out eval) vs 3–5 gold few-shot examples in-prompt — the model imitates Wally's actual voice, not generic-assistant voice
  • SKILL.md: role + SOP rules · versioned KB slices · gold + negative examples · grounding rule (every claim traces to a client-record field) · output schema · gate level
  • Eval to target: classification → exact-match (triage ≥95%); generative → rubric judge + pairwise vs the human original + the blind test
  • Ship at G2; production edits feed the precision-ledger; skill-evaluator regression before any version promotes

The two assets that can't be generated

The block library (human-designed components every site and page composes from — the AI arranges, never invents CSS) and the design-directions library (named, versioned taste recipes the design-director casts from). Designer-led sprints, build step 5.2 — and later, part of the licensed SaaS IP alongside the knowledge base. Model tiers: fast tier for high-volume classification; top tier for judgement skills; KB slices sit in the cached prompt prefix so per-run cost stays low.

The health score — computed, never set

health = 35% KPI vs benchmark + 20% open flags (severity × age) + 15% engagement (logins · report opens · attendance) + 10% SLA record + 10% sentiment + 10% billing
≥80 ● healthy · 60–79 ● watch · 40–59 ● attention · <40 ● at-risk → phase 7 — humans annotate, never override

Underneath: the entity matcher routes every event to the right record (email domain, phone line, Fathom participants, account-ID maps, portal auth; <0.8 confidence → human queue; unknown domain → prospect flow — never lost mail), and the Ads autonomy ladder (Tier A auto → Tier B trust-earned → Tier C human forever) plus the client editing tiers (structured → prompt-box → visual editor → contractual raw control) apply the same graduated-trust architecture to spend and to websites.
